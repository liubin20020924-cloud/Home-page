### 新用户（首次配置）

1. **第一步**: 阅读 [00 - 快速使用指南](./00-QUICK_START.md) ⭐
   - 快速了解 CI/CD 配置步骤
   - 按步骤完成 GitHub 和云主机配置
   - 验证配置并测试部署

2. **第二步**（可选）: 阅读 [01 - CI/CD 完整介绍](./01-INTRODUCTION.md)
   - 了解 CI/CD 系统架构
   - 理解工作流程
   - 熟悉核心组件

3. **第三步**: 按照 [02 - 配置总结和使用](./02-CONFIGURATION.md) 深入配置
   - GitHub 详细配置
   - 云主机详细配置
   - Secrets 和环境变量详细设置

4. **第四步**: 测试配置
   - 验证 GitHub Actions 能正常运行
   - 验证应用服务正常
   - 执行完整部署测试

## 💡 使用建议

### 首次配置

1. **按顺序阅读核心文档**
   - 01 → 02 → 03 → 04
   - 确保理解每个环节

2. **边读边实践**
   - 读完文档后立即实践
   - 记录遇到的问题
   - 建立知识库

3. **保存常用命令**
   - 将常用命令保存到笔记
   - 创建脚本简化操作

### 日常使用

1. **快速问题定位**
   - 遇到问题先查阅故障排除指南
   - 使用文档中的诊断命令
   - 记录解决方案供后续参考

2. **定期回顾文档**
   - 每月查看一次测试指南
   - 部署前查看部署历史
   - 系统更新后查看配置说明

3. **贡献文档改进**
   - 发现文档错误及时反馈
   - 补充遗漏的内容
   - 分享使用经验
