## 📝 配置步骤总结

### 第一步: GitHub配置

#### 1.1 生成SSH密钥对
```bash
ssh-keygen -t rsa -b 4096 -C "github-gitee-sync" -f ~/.ssh/github_gitee_rsa
```

#### 1.2 配置GitHub Secrets
访问: https://github.com/liubin20020924-cloud/Home-page/settings/secrets/actions

**Secret: SSH_PRIVATE_KEY**
```bash
# 复制私钥内容
cat ~/.ssh/github_gitee_rsa
```
粘贴完整的私钥(包括 `-----BEGIN OPENSSH PRIVATE KEY-----` 和 `-----END OPENSSH PRIVATE KEY-----`)

**Secret: GITEE_REPO**
```
liubin_studies/Home-page
```

### 第二步: Gitee配置

#### 2.1 添加SSH公钥
访问: https://gitee.com/profile/sshkeys

1. 点击"添加公钥"
2. 标题: `GitHub Sync Key`
3. 粘贴公钥内容:
```bash
cat ~/.ssh/github_gitee_rsa.pub
```
4. 点击"确定"

#### 2.2 测试SSH连接
```bash
ssh -T git@gitee.com
```

### 第三步: 云主机配置

#### 3.1 生成Webhook密钥
```bash
python3 -c "import secrets; print(secrets.token_urlsafe(32))"
```

#### 3.2 配置密钥文件
```bash
# 编辑webhook接收器
nano /opt/integrate-code/scripts/webhook_receiver.py
# 找到: WEBHOOK_SECRET = os.getenv('WEBHOOK_SECRET', 'your-webhook-secret-here')
# 替换为: WEBHOOK_SECRET = os.getenv('WEBHOOK_SECRET', '生成的密钥')

# 编辑自动检测脚本
nano /opt/integrate-code/scripts/check_and_deploy.sh
# 找到: WEBHOOK_SECRET="your-webhook-secret-here"
# 替换为: WEBHOOK_SECRET="生成的密钥"
```

#### 3.3 安装服务
```bash
bash /opt/integrate-code/scripts/deploy_service.sh
```

#### 3.4 验证服务
```bash
# 查看webhook服务
systemctl status webhook-receiver

# 查看定时任务
crontab -l

# 查看日志
tail -f /var/log/integrate-code/webhook.log
tail -f /var/log/integrate-code/auto-deploy.log
```

## ✨ 使用示例

### 开发新功能

```bash
# 1. 创建功能分支
git checkout -b feat/add-user-dashboard

# 2. 编写代码并提交
git add .
git commit -m "feat(home): 添加用户仪表盘功能"

# 3. 推送到GitHub
git push origin feat/add-user-dashboard

# 4. 在GitHub创建Pull Request

# 5. 合并后自动触发:
#    - GitHub Actions同步到Gitee
#    - 云主机自动部署
```

### 修复Bug

```bash
# 1. 创建修复分支
git checkout -b fix/login-error

# 2. 修复并提交
git commit -m "fix(auth): 修复登录验证错误"

# 3. 推送并创建PR
git push origin fix/login-error

# 4. 合并后自动部署
```

### 紧急修复

```bash
# 1. 从main创建hotfix分支
git checkout -b hotfix/critical-security-fix

# 2. 快速修复
git commit -m "hotfix(security): 修复安全漏洞"

# 3. 推送并快速合并
git push origin hotfix/critical-security-fix

# 4. 合并后立即部署
```

## 📊 提交规范参考

### 提交格式
```
<type>(<scope>): <subject>
```

### Type类型
- `feat`: 新功能
- `fix`: Bug修复
- `docs`: 文档更新
- `style`: 代码格式调整
- `refactor`: 代码重构
- `perf`: 性能优化
- `test`: 测试相关
- `chore`: 构建/工具链
- `revert`: 回退提交

### Scope范围
- `home`: 官网系统
- `kb`: 知识库系统
- `case`: 工单系统
- `auth`: 认证模块
- `db`: 数据库
- `api`: API接口
- `ui`: 前端界面
- `deploy`: 部署相关
- `config`: 配置文件

### 示例
```bash
feat(home): 添加产品对比功能
fix(auth): 修复Token过期问题
docs(kb): 更新知识库API文档
perf(db): 优化查询性能
```
