## 一、概述

### 1.1 系统说明

当新客户通过官网"联系我们"页面提交联系信息时，系统会自动：

1. **创建用户账户**：使用随机生成的密码
2. **发送通知邮件**：将登录信息发送到客户邮箱
3. **标记账户状态**：账户初始状态为"待审核"
4. **创建工单记录**：根据联系信息自动生成初始工单

### 1.2 功能优势

✅ **简化用户注册流程**：客户无需单独注册
✅ **自动化账户创建**：减少人工操作
✅ **工单自动关联**：账户与工单自动绑定
✅ **安全的信息传递**：通过邮件发送登录凭据

## 二、用户账户创建流程

### 2.1 客户端流程

#### 步骤1：提交联系信息

客户在官网首页填写联系表单：

```
公司名称：[必填]
联系人姓名：[必填]
联系邮箱：[必填]
联系电话：[必填]
产品：[必填]
问题类型：[必填]
留言内容：[必填]
```

#### 步骤2：系统自动处理

系统后台自动执行：

1. 验证邮箱格式
2. 检查邮箱是否已注册
3. 生成随机密码（12位，包含大小写字母、数字、特殊字符）
4. 创建用户账户，角色为"customer"
5. 发送通知邮件到联系邮箱

#### 步骤3：查收邮件

客户会收到一封邮件，标题为：

```
【云户科技】新客户注册申请 - [公司名称]
```

邮件内容包含：

**选项A：如果用户账户已创建**

```
📋 新客户注册申请

公司名称：[填写的公司名称]

留言内容：
[客户填写的留言内容]

🔑 客户登录信息（请通过邮件发送给客户）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

用户名：[自动生成的用户名]
初始密码：[自动生成的随机密码]

⚠️ 请将此信息通过邮件发送给客户，提醒客户首次登录后修改密码
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**选项B：如果用户账户未创建**

```
📋 新客户注册申请

公司名称：[填写的公司名称]

留言内容：
[客户填写的留言内容]

账户状态：待审核（需人工处理）
```

#### 步骤4：首次登录

**场景A：账户已创建**

1. 访问登录页面：`https://yunhukeji.com/case/login`
2. 输入用户名和初始密码
3. 登录成功
4. 系统提示修改密码
5. 进入"个人设置"修改初始密码

**场景B：账户待审核**

1. 管理员审核客户申请
2. 通过后台创建用户账户
3. 通知客户账户已创建
4. 客户按照上述登录流程进行首次登录

### 2.2 系统后台流程

#### 自动处理流程（代码逻辑）

```python
# routes/home_bp.py - contact() 函数

1. 接收并验证联系表单数据
   - 公司名称（必填）
   - 联系人姓名（必填）
   - 联系邮箱（必填）
   - 联系电话（必填）
   - 产品（必填）
   - 问题类型（必填）
   - 留言内容（必填）

2. 检查邮箱是否已注册
   - 如果邮箱已存在 → 返回提示信息
   - 如果邮箱不存在 → 继续创建账户

3. 生成随机密码（12位）
   - 包含大小写字母、数字、特殊字符
   - 示例：aB3$xK9!mP2

4. 创建用户账户
   - 用户名：使用联系邮箱作为用户名
   - 密码：使用生成的随机密码
   - 角色：customer
   - 状态：pending（待审核）或active（如果启用自动激活）

5. 发送通知邮件
   - 邮件主题：【云户科技】新客户注册申请 - [公司名称]
   - 邮件内容：包含登录信息或审核状态

6. 创建初始工单
   - 根据联系信息自动创建工单
   - 工单状态：pending（待处理）

7. 返回成功响应
   - 向客户显示"留言提交成功"提示
```

## 三、用户类型说明

### 3.1 用户角色

系统支持以下用户角色：

| 角色 | 说明 | 权限 | 账户创建方式 |
|------|------|--------|----------------|
| **admin** | 系统管理员 | 完全权限，可管理所有系统和用户 | 手动创建（后台） |
| **user** | 内部员工 | 工单管理、知识库管理 | 手动创建（后台） |
| **customer** | 客户用户 | 提交工单、查看自己工单 | 通过联系表单自动创建 |

### 3.2 客户用户权限

**登录后可以访问**：

✅ **首页**：查看公司介绍和服务
✅ **知识库**：浏览技术文档和常见问题
✅ **提交工单**：创建新的服务请求
✅ **我的工单**：查看自己提交的所有工单
✅ **工单详情**：查看工单详情、回复消息、提交满意度评价

**无法访问**：

❌ 工单管理：管理所有工单（仅 admin/user 可见）
❌ 统计报表：查看系统统计（仅 admin/user 可见）
❌ 用户管理：管理系统用户（仅 admin 可见）

## 四、登录信息说明

### 4.1 账户类型说明

#### 场景1：自动创建的临时账户

**账户特点**：
- 用户名：联系邮箱
- 密码：系统自动生成的12位随机密码
- 状态：初始为 `pending`（待审核）或 `active`（根据配置）
- 角色：`customer`（客户）
- 需要审核：是（默认）或 否（如果启用自动激活）

**审核状态说明**：

```
状态：pending（待审核）
含义：账户已创建但需要管理员审核才能使用
处理：管理员在后台用户管理页面审核用户
激活：管理员激活后，客户可以正常登录
```

```
状态：active（已激活）
含义：账户已创建并可立即使用
处理：客户可以立即使用提供的用户名和密码登录
推荐：登录后立即修改密码
```

### 4.2 登录方式

#### 方式一：使用自动生成的凭据

1. 打开登录页面：`https://yunhukeji.com/case/login`
2. 输入用户名：联系邮箱
3. 输入密码：邮件中提供的初始密码
4. 点击"登录"按钮

#### 方式二：修改密码后登录

1. 首次登录成功后，进入"个人设置"
2. 点击"修改密码"
3. 输入初始密码和新密码
4. 修改成功
5. 下次使用新密码登录

### 4.3 密码重置

**场景**：忘记密码或密码已修改

**操作流程**：

1. 访问登录页面
2. 点击"忘记密码"
3. 输入注册邮箱
4. 系统发送密码重置链接到邮箱
5. 点击重置链接，设置新密码
6. 使用新密码登录

**注意**：
- 密码重置链接有效期为24小时
- 重置后需要重新登录
- 建议使用强密码（见第五章）

## 五、密码安全建议

### 5.1 初始密码

**特点**：
- 系统自动生成
- 长度：12位
- 复杂度：包含大小写字母、数字、特殊字符
- 示例：`aB3$xK9!mP2`

**安全性**：
- ✅ 满足强密码要求
- ✅ 唯一且随机
- ⚠️ **建议首次登录后立即修改**

### 5.2 密码安全建议

#### 推荐做法

1. **立即修改初始密码**
   - 首次登录后立即修改
   - 设置自己记住的密码
   - 避免多次登录才修改

2. **设置强密码**
   - 长度至少8位（推荐12位以上）
   - 包含大小写字母、数字、特殊字符
   - 避免使用个人信息（生日、姓名等）
   - 不要与其他网站使用相同密码

3. **定期更换密码**
   - 建议每3-6个月更换一次
   - 重要账户单独管理

4. **保护密码信息**
   - 不要在邮件中保存密码
   - 不要截图保存密码
   - 不要在浏览器中保存密码（公共设备）
   - 不要与第三方分享密码

#### 密码强度示例

**弱密码（不推荐）**：
```
❌ 123456
❌ password
❌ abc123
❌ company2024
```

**中等级别（勉强可用）**：
```
⚠️ Com@ny123
⚠️ Password1!
⚠️ User@2024
```

**强密码（推荐）**：
```
✅ aB3$xK9!mP2
✅ Kj#7dL@xP9$mQ
✅ 9xM$nR2@kL#8sP
```

### 5.3 账户安全最佳实践

1. **启用两步验证**（可选）
   - 系统未来可支持短信或邮箱验证
   - 提供额外的安全保护层

2. **定期检查登录记录**
   - 在"个人设置"中查看最近登录记录
   - 发现异常登录立即修改密码
   - 联系技术支持处理

3. **安全退出登录**
   - 使用官方"退出"按钮
   - 不要直接关闭浏览器
   - 公共设备务必退出

4. **保护邮箱安全**
   - 启用邮箱两步验证
   - 定期检查邮箱安全设置
   - 不要泄露邮箱登录凭据

## 六、常见问题

### 6.1 客户常见问题

#### Q1：为什么需要自动创建账户？

**A**：
- 简化客户注册流程
- 客户无需单独注册
- 账户与工单自动关联
- 提供更好的用户体验

#### Q2：我没有收到邮件怎么办？

**A**：按以下步骤排查：

1. **检查垃圾邮件文件夹**
   - 邮件可能被误判为垃圾邮件
   - 搜索邮件主题：【云户科技】

2. **检查邮箱地址是否正确**
   - 确认联系表单填写的邮箱地址正确
   - 检查是否有拼写错误

3. **等待几分钟**
   - 邮件发送可能需要1-5分钟
   - 如果10分钟后仍未收到，请联系技术支持

4. **检查邮箱设置**
   - 确认邮箱未阻止外部邮件
   - 检查垃圾邮件过滤设置

#### Q3：初始密码太复杂记不住怎么办？

**A**：

1. 首次登录后立即修改密码
2. 使用密码管理器（如1Password、LastPass等）
3. 如果忘记密码，使用"忘记密码"功能重置
4. 联系技术支持，可以协助重置密码

#### Q4：账户一直显示"待审核"怎么办？

**A**：

1. 等待管理员审核（通常1-2个工作日）
2. 可以通过电话联系技术支持催促审核
3. 审核通过后会收到通知邮件
4. 如果超过3个工作日未处理，请联系技术支持

#### Q5：登录后能看到哪些信息？

**A**：作为客户用户登录后可以访问：

✅ **首页**：公司介绍和服务内容
✅ **知识库**：技术文档、常见问题
✅ **提交工单**：创建新的服务请求
✅ **我的工单**：查看自己提交的所有工单
✅ **工单详情**：查看工单进度、回复消息、提交满意度评价

❌ **不能访问**：
- 管理功能（工单管理、统计报表、用户管理）
- 其他用户的信息
- 系统配置

#### Q6：可以创建多个账户吗？

**A**：

- 每个邮箱地址只能创建一个账户
- 如果需要多个人员访问，建议：
  1. 创建公司主账户
  2. 在知识库中分享访问权限（未来功能）
  3. 或联系技术支持创建员工账户

#### Q7：忘记用户名或密码怎么办？

**A**：

**忘记用户名**：
- 用户名就是您提交联系表单时填写的邮箱地址
- 可以查看邮件记录确认

**忘记密码**：
1. 访问登录页面：`https://yunhukeji.com/case/login`
2. 点击"忘记密码"链接
3. 输入注册邮箱
4. 系统发送密码重置链接到邮箱
5. 点击链接设置新密码
6. 使用新密码登录

#### Q8：密码修改后原来的密码还能用吗？

**A**：
- **不能**。密码修改后立即生效
- 旧的密码立即失效
- 建议将修改后的密码妥善保存
- 如果忘记，需要通过"忘记密码"功能重置

### 6.2 技术支持常见问题

#### Q9：客户说没有收到邮件，如何处理？

**A**：按以下步骤排查

1. **确认邮箱地址正确**
   - 查看系统日志中的邮件收件人
   - 确认是否为客户填写的正确邮箱

2. **检查邮件配置**
   - 确认 SMTP 服务器配置正确
   - 检查邮件服务器连接状态
   - 查看邮件发送日志

3. **查看发送日志**
   ```bash
   # 查看应用日志
   tail -f logs/app.log | grep "邮件发送"
   
   # 或使用 journalctl
   journalctl -u your-service -f | grep "邮件"
   ```

4. **重新发送邮件**
   - 在后台用户管理页面找到该用户
   - 点击"重发激活邮件"
   - 或手动告知客户登录信息

#### Q10：如何查看账户创建状态？

**A**：

**方法一：查看系统日志**
```bash
# 查看用户创建日志
tail -f logs/app.log | grep "创建用户"
```

**方法二：在后台用户管理页面查看**
1. 登录管理员账户
2. 进入"用户管理"
3. 查看用户列表
4. 查看用户状态（pending/active）

**方法三：数据库查询**
```sql
-- 查看用户状态
SELECT id, username, email, role, status, create_time 
FROM users 
ORDER BY create_time DESC 
LIMIT 20;

-- 查看某个特定用户
SELECT * FROM users WHERE email = 'customer@example.com';
```

## 七、管理员操作指南

### 7.1 账户审核

#### 审核流程

**场景**：客户提交联系信息后，需要管理员审核账户

**操作步骤**：

1. **登录管理员账户**
   - 访问：`https://yunhukeji.com/user-mgmt/login`
   - 使用管理员账户登录

2. **进入用户管理页面**
   - 点击"用户管理"菜单
   - 查看用户列表

3. **查看待审核用户**
   - 筛选条件：状态 = pending
   - 查看用户详细信息

4. **审核用户**
   
   **选项A：激活用户**
   - 点击用户操作"激活"
   - 用户状态变为 active
   - 系统发送激活通知邮件
   
   **选项B：拒绝用户**
   - 点击用户操作"拒绝"
   - 用户状态变为 rejected
   - 可填写拒绝原因
   - 系统发送拒绝通知邮件

5. **通知客户**
   - 审核结果会自动发送邮件给客户
   - 也可以通过电话联系客户

#### 审核标准

**通过审核的标准**：
- ✅ 联系信息完整且真实
- ✅ 邮箱地址有效
- ✅ 公司信息可验证
- ✅ 符合客户资质要求

**拒绝审核的标准**：
- ❌ 邮箱地址无效或不存在
- ❌ 公司信息不完整
- ❌ 联系信息无法验证
- ❌ 不符合客户资质要求
- ❌ 联系方式无效

### 7.2 用户管理

#### 查看用户列表

**字段说明**：
- 用户名：登录邮箱
- 显示名称：联系人姓名
- 邮箱：联系邮箱
- 角色：admin/user/customer
- 状态：pending/active/rejected/blocked
- 创建时间：账户创建时间
- 最后登录：最近一次登录时间

**筛选功能**：
- 按角色筛选：admin/user/customer
- 按状态筛选：pending/active/rejected
- 按创建时间筛选：最近7天/30天/90天
- 搜索用户：按邮箱或显示名称搜索

#### 用户操作

**激活用户**：
- 状态从 pending → active
- 发送激活邮件通知

**重置密码**：
- 生成新的随机密码
- 发送密码到用户邮箱
- 要求用户首次登录后修改

**编辑用户**：
- 修改显示名称
- 修改角色
- 修改公司信息

**禁用用户**：
- 状态从 active → blocked
- 用户无法登录
- 保留用户数据和工单历史

**删除用户**：
- ⚠️ 谨慎操作，不可恢复
- 同时删除相关工单和消息
- 建议禁用而不是删除

### 7.3 手动创建用户

#### 适用场景

1. 客户通过电话或邮件联系，要求创建账户
2. 企业内部员工需要访问系统
3. 测试或演示账户创建
4. 特殊用户类型（合作伙伴、供应商等）

#### 操作步骤

1. **进入用户管理页面**
   - 登录管理员账户
   - 点击"新建用户"按钮

2. **填写用户信息**
   ```
   用户名：[邮箱地址，唯一]
   显示名称：[显示名称]
   密码：[临时密码或自动生成]
   确认密码：[重复密码]
   角色：[customer/user/admin]
   邮箱：[邮箱地址]
   ```

```python
# routes/home_bp.py - contact() 函数简化流程

def contact():
    # 1. 接收表单数据
    data = request.get_json()
    
    # 2. 验证必填字段
    if not all([
        data.get('company_name'),
        data.get('contact_name'),
        data.get('contact_email'),
        data.get('contact_phone'),
        data.get('product'),
        data.get('issue_type'),
        data.get('message')
    ]):
        return error_response('请填写完整信息', 400)
    
    # 3. 检查邮箱是否已存在
    cursor.execute("SELECT id FROM users WHERE email = %s", 
                 (data['contact_email'],))
    existing_user = cursor.fetchone()
    
    if existing_user and existing_user['role'] != 'customer':
        return error_response('该邮箱已被注册', 400)
    
    # 4. 生成随机密码
    import secrets
    import string
    password_chars = string.ascii_letters + string.digits + '!@#$%^&*'
    random_password = ''.join(secrets.choice(password_chars) for _ in range(12))
```

## 九、安全注意事项

### 9.1 敏感信息处理

#### 邮件发送要求

- ✅ 邮件内容包含密码时，应该使用安全传输
- ✅ 避免在邮件正文中明文显示密码
- ✅ 建议使用临时链接或独立页面传递密码

#### 密码存储

- ✅ 使用 PBKDF2 加密存储密码
- ✅ 永远不要明文存储或记录密码
- ✅ 定期更换加密算法（PBKDF2 → Argon2）

### 9.2 防护措施

#### 防止恶意注册

- 同一IP限制：短时间内只能提交一定数量的表单
- 邮箱验证：邮箱需要验证有效性
- 内容过滤：过滤垃圾内容和恶意输入
- 验证码：未来可添加图形验证码

#### 防止暴力破解

- 登录失败次数限制：5次失败后锁定账户15分钟
- IP 限制：同一IP短时间内只能尝试一定次数登录
- 密码复杂度要求：强制使用强密码
- 两步验证：重要账户建议启用
