## 1. 系统概述

工单系统是云户科技企业官网的重要组成部分，用于客户提交技术问题、服务请求和产品咨询，并与客服人员进行在线交流。系统采用论坛帖子/回复模式，便于问题跟踪与归档。

### 系统特点

- 🎯 **论坛式交流**：主帖与回复结构清晰，历史记录完整
- 👥 **角色权限分明**：区分客户、普通用户（客服）、管理员
- 🔍 **智能查询**：根据用户角色动态显示工单列表
- 📝 **完整记录**：保存所有工单和回复历史
- 🎨 **现代界面**：响应式设计，支持多设备访问
- 📧 **邮件通知**：关键事件自动邮件通知相关人员
- ⭐ **满意度评价**：工单完成后邀请客户评价服务
- 📊 **数据报表**：提供多维度统计分析与导出

### 技术架构

| 层次 | 技术栈 | 说明 |
|------|--------|------|
| **前端** | Bootstrap 5.3.0 + JavaScript | 论坛式卡片界面 |
| **后端** | Flask 3.0.3 + Python 3.8+ | RESTful API 路由 |
| **数据库** | MySQL/MariaDB | 独立数据库 casedb |
| **邮件服务** | SMTP (企业微信邮箱) | 自动通知 |

## 2. 核心功能

### 2.1 提交工单
客户或内部员工均可提交工单，填写基本信息、选择产品/问题类型/优先级、描述问题并上传附件，提交后自动生成唯一工单编号。

### 2.2 我的工单
根据用户角色展示不同的工单列表，支持按状态、优先级筛选及关键词搜索。

- **管理员/普通用户**：显示所有工单
- **客户**：显示联系人为自己的工单

### 2.3 工单详情
查看工单完整信息、所有回复，并进行回复操作。界面区分客户回复与客服回复。

### 2.4 回复与状态更新
客服可回复工单并更新状态（待处理、处理中、已完成、已关闭）；客户仅可回复，不可修改状态。

### 2.5 工单管理（管理员）
管理员可查看所有工单、分配处理人、更新状态，拥有全部管理权限。

### 2.6 满意度评价
工单完成后触发客户评价，收集服务反馈。

### 2.7 工单统计
为管理员提供可视化数据仪表盘，支持指标统计与导出。

## 3. 用户角色

### 角色定义

| 角色 | 英文标识 | 权限说明 |
|------|---------|---------|
| **管理员** | admin | 最高权限：查看所有工单、分配处理人、修改状态、管理用户、查看报表 |
| **普通用户** | user | 公司内部客服/技术支持：查看所有工单、回复、更新状态（无权分配） |
| **客户** | customer | 提交工单、查看自己的工单、回复（不能修改状态） |

### 角色功能对比

| 功能 | 管理员 | 普通用户 | 客户 |
|------|-------|---------|------|
| 提交工单 | ✅ | ✅ | ✅ |
| 查看工单列表 | 所有工单 | 所有工单 | 仅自己的工单 |
| 查看任意工单详情 | ✅ | ✅ | ❌（只能查看自己的） |
| 回复工单 | ✅ | ✅ | ✅ |
| 更新工单状态 | ✅ | ✅ | ❌ |
| 分配工单处理人 | ✅ | ❌ | ❌ |
| 管理用户 | ✅ | ❌ | ❌ |
| 查看报表 | ✅ | ❌ | ❌ |

### 角色说明

#### 管理员 (admin)
- **主要职责**：系统管理、用户管理、工单分配
- **权限范围**：查看所有工单、修改工单状态、管理用户账号、配置系统参数
- **典型用户**：技术总监、系统管理员

#### 普通用户 (user)
- **主要职责**：处理理工单、提供技术支持
- **权限范围**：查看所有工单、回复客户、更新工单状态
- **典型用户**：技术支持工程师、客服代表

#### 客户 (customer)
- **主要职责**：提交工单、描述问题、与客服沟通
- **权限范围**：提交工单、查看自己的工单、回复工单
- **典型用户**：企业客户、个人用户

## 4. 数据表结构

采用独立数据库 `casedb`，核心表为 tickets（工单表）和 messages（回复表），新增 satisfaction 表（满意度评价表）。

### 4.1 tickets 表（工单表）

| 字段名 | 类型 | 说明 | 约束 |
|--------|------|------|------|
| `id` | INT | 主键ID | 自增，PRIMARY KEY |
| `ticket_id` | VARCHAR(32) | 工单编号 | UNIQUE, NOT NULL |
| `customer_name` | VARCHAR(100) | 客户公司/个人名称 | NOT NULL |
| `customer_contact_name` | VARCHAR(100) | 联系人姓名 | NOT NULL |
| `customer_contact` | VARCHAR(50) | 联系电话 | - |
| `customer_email` | VARCHAR(100) | 客户邮箱 | - |
| `cc_emails` | TEXT | 抄送邮箱（逗号分隔） | - |
| `submit_user` | VARCHAR(100) | 提交用户（customer username） | - |
| `product` | VARCHAR(50) | 涉及产品 | - |
| `issue_type` | VARCHAR(20) | 问题类型 | - |
| `priority` | VARCHAR(10) | 优先级 | - |
| `title` | VARCHAR(200) | 工单标题 | NOT NULL |
| `content` | TEXT | 工单详细描述 | - |
| `resolution` | TEXT | 解决方案 | - |
| `status` | VARCHAR(10) | 工单状态 | 默认: 'pending' |
| `assignee` | VARCHAR(100) | 处理人 | - |
| `create_time` | DATETIME | 创建时间 | 默认: NOW() |
| `update_time` | DATETIME | 更新时间 | 默认: NOW() |

**状态值**：
- `pending`: 待处理
- `processing`: 处理中
- `completed`: 已完成
- `closed`: 已关闭

**优先级值**：
- `urgent`: 紧急
- `high`: 高
- `medium`: 中
- `low`: 低

**问题类型值**：
- `technical`: 技术支持
- `feature`: 功能咨询
- `bug`: Bug报告
- `other`: 其他

### 4.2 messages 表（回复表）

| 字段名 | 类型 | 说明 | 约束 |
|--------|------|------|------|
| `id` | INT | 主键ID | 自增，PRIMARY KEY |
| `ticket_id` | VARCHAR(32) | 关联的工单编号 | NOT NULL, FOREIGN KEY |
| `sender` | VARCHAR(20) | 发送者类型（customer 或 service） | NOT NULL |
| `sender_name` | VARCHAR(100) | 发送者名称 | - |
| `content` | TEXT | 回复内容 | NOT NULL |
| `send_time` | DATETIME | 发送时间 | 默认: NOW() |

**发送者类型**：
- `customer`: 客户回复
- `service`: 客服回复

### 4.3 satisfaction 表（满意度评价表）

| 字段名 | 类型 | 说明 | 约束 |
|--------|------|------|------|
| `id` | INT | 主键ID | 自增，PRIMARY KEY |
| `ticket_id` | VARCHAR(32) | 关联的工单编号 | UNIQUE, FOREIGN KEY |
| `rating` | TINYINT | 评分（1-5星） | NOT NULL |
| `comment` | TEXT | 评价文字 | - |
| `create_time` | DATETIME | 评价时间 | 默认: NOW() |

### 4.4 数据表关系

```
users (用户表 - 统一用户管理)
    ↓ (1:N) (提交用户)
tickets (工单表)
    ├─ submit_user → users.username
    ├─ customer_contact → users.username  (用于客户查询自己的工单)
    ├─ customer_email → users.email        (用于客户查询自己的工单)
    ├─ assignee → users.username             (处理人)
    ↓ (1:N)
messages (回复表)
    └─ ticket_id → tickets.ticket_id
    ↓ (1:1)
satisfaction (满意度表)
    └─ ticket_id → tickets.ticket_id
```

### 4.5 索引说明

**tickets 表索引**：
```sql
PRIMARY KEY (id)
UNIQUE KEY (ticket_id)
INDEX idx_status (status)
INDEX idx_priority (priority)
INDEX idx_submit_user (submit_user)
INDEX idx_customer_contact (customer_contact)
INDEX idx_customer_email (customer_email)
INDEX idx_create_time (create_time)
```

**messages 表索引**：
```sql
PRIMARY KEY (id)
INDEX idx_ticket_id (ticket_id)
INDEX idx_send_time (send_time)
FOREIGN KEY (ticket_id) REFERENCES tickets(ticket_id) ON DELETE CASCADE
```

**satisfaction 表索引**：
```sql
PRIMARY KEY (id)
UNIQUE KEY (ticket_id)
FOREIGN KEY (ticket_id) REFERENCES tickets(ticket_id) ON DELETE CASCADE
```

## 5. 功能模块说明

### 5.1 提交工单模块

**访问路径**：`/case/submit`

**流程**：
```
用户登录后访问提交页面
    ↓
填写表单（客户名称、联系人、联系方式、产品、问题类型、优先级、标题、内容、附件）
    ↓
提交表单
    ↓
生成工单编号（TK-{YYYYMMDDHHMMSS}-{6位随机字符}）
    ↓
系统自动发送邮件通知所有管理员及指定客服邮箱
    ↓
跳转至"我的工单"列表
```

**表单验证**：
- 客户名称、联系人、联系方式、标题、内容必填
- 邮箱格式验证
- 附件文件大小检查（单文件上限20MB）

### 5.2 工单查询模块

**访问路径**：`/case/my-tickets`

**查询逻辑**：
```python
# 管理员/普通用户：显示所有工单
SELECT * FROM tickets ORDER BY create_time DESC

# 客户：显示联系人为自己的工单
SELECT * FROM tickets 
WHERE (customer_contact = 'username' OR customer_email = 'username')
ORDER BY create_time DESC
```

**筛选功能**：
- **状态筛选**：pending/processing/completed/closed
- **优先级筛选**：urgent/high/medium/low
- **关键词搜索**：工单编号或标题模糊匹配
- **分页**：默认每页10条

### 5.3 工单详情模块

**访问路径**：`/case/ticket/<ticket_id>`

**页面结构**：
```
┌─────────────────────────────────────────────┐
│ 工单详情                                   │
├─────────────────────────────────────────────┤
│ 顶部：工单标题、编号、创建时间          │
│       状态标签、优先级标签                 │
├─────────────────────────────────────────────┤
│ 主帖区（工单描述）                      │
│ - 客户头像（姓名首字母）                 │
│ - 客户名称和角色                         │
│ - 联系方式（电话、邮箱）               │
│ - 详细描述                               │
├─────────────────────────────────────────────┤
│ 回复区                                   │
│ - 回复列表（按时间倒序）               │
│ - 客服回复：绿色边框                   │
│ - 客户回复：蓝色边框                   │
├─────────────────────────────────────────────┤
│ 回复输入框                               │
│ - 多行文本输入                           │
│ - 发送按钮                               │
└─────────────────────────────────────────────┘
```

**加载逻辑**：
```javascript
// 页面加载时
1. 调用 GET /case/api/ticket/<ticket_id> 获取工单详情
2. 调用 GET /case/api/ticket/<ticket_id>/messages 获取回复列表
3. 渲染页面内容

// 用户发送回复
1. 调用 POST /case/api/ticket/<ticket_id>/message 发送回复
2. 重新加载回复列表
3. 自动滚动到最新回复
```

### 5.4 回复功能模块

**接口**：`POST /case/api/ticket/<ticket_id>/message`

**逻辑**：
```python
# 根据当前用户角色确定 sender 类型
sender = 'customer' if session.get('role') == 'customer' else 'service'
sender_name = session.get('display_name') or session.get('username', '匿名用户')

# 插入回复记录
INSERT INTO messages (ticket_id, sender, sender_name, content, send_time)
VALUES (?, ?, ?, ?, NOW())
```

### 5.5 工单管理模块（管理员）

**访问路径**：`/case/admin/tickets`

**功能**：
- 查看所有工单列表（支持筛选、搜索）
- 批量或单个更新工单状态
- 分配处理人（从现有用户中选择）
- 查看工单统计概览（可链接至报表模块）

### 5.6 满意度评价模块

**触发机制（优先级顺序）**：
1. **页面弹窗**：当工单状态变更为"已完成"或"已关闭"时，若客户当前在线，立即弹出评价窗口
2. **邮件邀请**：若客户未在弹窗中评价（或未在线），系统发送评价邀请邮件，邮件包含专属链接

**评价入口**：
- 若前两种方式均未完成评价，工单详情页底部保留"评价"按钮，客户可随时进入评价（仅一次）

**评价形式**：
- **评分方式**：满意 / 一般 / 不满意（对应表情图标）或 1-5 星评分
- **文字评价框**：选填

**可见性**：
- 评价内容对客服匿名显示（仅显示评分和文字，不显示评价人）
- 管理员可查看所有评价及对应的客服人员（用于绩效考核）

### 5.7 工单统计模块

**访问路径**：`/case/admin/reports`

**仪表盘概览（数字卡片）**：
- 工单总量（总提交数、未处理数、处理中数、已完成数、已关闭数）
- 按优先级分布（紧急/高/中/低）
- 按问题类型分布（技术支持/功能咨询/Bug报告/其他）
- 按产品分布

**图表展示**：
- 采用数字卡片形式展示核心指标
- 支持时间筛选：今日 / 本周 / 本月 / 自定义时间段
- 可切换为柱状图/饼图查看分布（后续迭代）

**导出功能**：
- 支持将当前筛选条件下的报表数据导出为 Excel/CSV 文件
- 导出字段包括：工单编号、客户名称、联系人、状态、优先级、创建时间、完成时间等

- **一致性强**：遵循统一的视觉规范

## 7. 通知机制

系统基于 SMTP 邮件服务发送通知，已配置完成。

### 触发事件及收件人

| 触发事件 | 收件人 |
|---------|--------|
| 新工单提交 | 所有管理员 + 指定客服邮箱 |
| 工单状态变更 | 工单提交客户 |
| 新回复（客户或客服） | 工单相关人员（客户、处理人、抄送人） |
| 工单分配给新的处理人 | 新的处理人 |
| 满意度评价邀请 | 工单提交客户（若未在弹窗中评价） |

### 邮件内容

邮件内容包括：
- 工单编号、标题
- 当前状态
- 变更/回复内容
- 直达工单详情页的链接

### 邮件配置

**当前配置**：企业微信邮箱（smtp.exmail.qq.com）
- **SMTP服务器**: smtp.exmail.qq.com
- **端口**: 465（SSL）
- **认证**: 使用邮箱密码

## 9. 常见问题

### Q1: 客户为什么看不到某些工单？

**A**: 客户只能看到联系人为自己的工单。系统会根据以下条件匹配：
- `customer_contact` = 客户的用户名
- 或 `customer_email` = 客户的邮箱

如果客户看不到工单，请检查：
1. 工单的联系人信息是否正确
2. 客户的用户名和邮箱是否正确

### Q2: 如何修改工单状态？

**A**: 工单状态只能由客服或管理员修改：
1. 进入工单详情页
2. 管理员可以点击"更新状态"按钮
3. 或通过 API 调用 `PUT /case/api/ticket/<ticket_id>/status`

### Q3: 工单支持哪些附件格式？

**A**: 支持所有文件类型，无限制。但单个文件大小不超过 20MB。

### Q4: 回复支持富文本吗？

**A**: 当前版本仅支持纯文本。未来版本将支持：
- Markdown 格式
- 代码高亮
- 图片上传

### Q5: 如何删除工单或回复？

**A**: 当前版本不支持删除功能。如需删除，请联系管理员直接操作数据库。

### Q6: 工单有大小限制吗？

**A**: 工单本身没有大小限制，但：
- 单次回复内容建议不超过 5000 字
- 单个附件大小限制为 20MB
- 每个工单最多上传 5 个附件

### Q7: 可以@提及其他用户吗？

**A**: 当前版本不支持。未来版本将支持在回复中@提及其他用户，并发送通知。

### Q8: 如何查看工单处理历史？

**A**: 工单详情页的回复列表就是完整的处理历史，按时间倒序显示所有交互记录。

### Q9: 客服如何知道有新工单？

**A**: 系统会自动发送邮件通知。客服也可以定期刷新"我的工单"页面查看最新工单。

### Q10: 工单会自动关闭吗？

**A**: 不会。工单需要客服或管理员手动关闭。建议设置工单 SLA（服务级别协议）规则，超时自动提醒。

### Q11: 满意度评价是强制的吗？

**A**: 不是。客户可以选择评价或不评价。系统通过弹窗和邮件两种方式邀请评价，客户可随时通过详情页的评价按钮进行评价（仅限一次）。

### Q12: 评价内容是公开的吗？

**A**: 评价内容对客服匿名显示，仅显示评分和文字，不显示评价人。管理员可以查看所有评价及对应的客服人员（用于绩效考核）。

### Q13: 如何导出报表数据？

**A**: 访问 `/case/admin/reports` 页面，在仪表盘下方找到"导出"按钮，支持导出为 Excel 或 CSV 格式。导出的数据会应用当前的筛选条件。
